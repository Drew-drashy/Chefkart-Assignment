# Chefkraft Server - Backend

This repository contains the backend for the Chefkraft application. It uses **Node.js** and **Sequelize** to manage a MySQL database for managing users and posts.

---

## Features

* Create and manage users and their posts.
* RESTful API endpoints for CRUD operations.
* Sequelize ORM for database interaction.
* MySQL as the database.
* Environment variable configuration for database credentials.

---

## Requirements

* Node.js (v16+)
* MySQL (v8.0+)
* npm or yarn

## API Endpoints

### User Routes

1. **Get All Users**
   * **GET** `/users`
   * Returns a list of all users.
2. **Get All Posts of a User**
   * **GET** `/users/:userId/posts`
   * Returns all posts created by a specific user.

### Post Routes

1. **Get All Posts**
   * **GET** `/posts`
   * Returns a list of all posts.
2. **Create a Post**
   * **POST** `/users/:userId/posts`
   * Creates a new post for a user.
3. **Update a Post**
   * **PUT** `/users/:userId/posts/:postId`
   * Updates a post for a specific user.
4. **Delete a Post**
   * **DELETE** `/users/:userId/posts/:postId`
   * Deletes a post created by a user.

## Database Schema

### Users Table

| Column            | Type         | Description                      |
| ----------------- | ------------ | -------------------------------- |
| `id`            | INT          | Auto-increment primary key.      |
| `name`          | VARCHAR(256) | User's name.                     |
| `mobile_number` | BIGINT       | Unique mobile number.            |
| `address`       | TEXT         | User's address.                  |
| `post_count`    | INT          | Number of posts by the user.     |
| `createdAt`     | DATETIME     | Auto-managed creation timestamp. |
| `updatedAt`     | DATETIME     | Auto-managed update timestamp.   |

### Posts Table

| Column          | Type     | Description                            |
| --------------- | -------- | -------------------------------------- |
| `id`          | INT      | Auto-increment primary key.            |
| `title`       | TEXT     | Title of the post.                     |
| `description` | TEXT     | Content of the post.                   |
| `userId`      | INT      | Foreign key referencing `users(id)`. |
| `images`      | JSON     | JSON array of image URLs.              |
| `createdAt`   | DATETIME | Auto-managed creation timestamp.       |
| `updatedAt`   | DATETIME | Auto-managed update timestamp.         |

### Using Postman

1. Install [Postman](https://www.postman.com/).
2. Use the API endpoints listed above.
3. Set headers:
   * Content-Type: `application/json`
4. Test CRUD operations for users and posts.

## Common Issues

### 1. Access Denied for MySQL

* Ensure the `.env` file contains correct database credentials.
* Verify MySQL is running and accessible on `localhost:3306`.

### 2. Sequelize Errors

* Ensure the database schema matches the models in the code.
* Use `sequelize.sync()` to create missing tables.
